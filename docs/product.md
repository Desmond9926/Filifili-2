# Product Requirement Document

## 1. Product Overview

本项目是一个类似 bilibili 的视频网站 MVP，面向普通用户与内容创作者，支持视频上传、视频播放、评论互动、个人主页、首页推荐与后台管理等核心功能。

当前阶段目标不是完整复刻成熟视频平台，而是优先搭建一个可用、可演示、可持续迭代的内容平台基础版本，完成“用户注册登录 → 投稿上传 → 转码处理 → 发布审核 → 视频播放 → 评论互动 → 后台管理”的闭环。

---

## 2. Product Goals

### 2.1 Main Goals
本项目 MVP 的目标包括：

1. 支持用户注册与登录
2. 支持创作者上传视频
3. 支持视频转码与播放
4. 支持基础互动功能（评论 / 点赞 / 收藏）
5. 支持用户个人主页
6. 支持首页推荐与分类浏览
7. 支持管理后台进行内容管理

### 2.2 Product Principles
- 优先打通核心闭环，不追求一次做全
- 优先保证体验完整，而不是功能堆叠
- 优先保证内容消费路径顺畅
- 优先保证投稿与管理流程稳定
- 所有功能围绕 MVP 可交付目标建设

## 3. Target Users

## 3.1 User Types

### 1) Guest
未登录访客，可以：
- 浏览首页
- 浏览视频详情页
- 播放公开视频
- 查看评论
- 查看用户主页

不能：
- 评论
- 点赞
- 收藏
- 上传视频
- 进入后台

### 2) Registered User
已注册登录用户，可以：
- 浏览视频
- 评论
- 点赞
- 收藏
- 查看和编辑个人资料

### 3) Creator
具备投稿能力的用户，可以：
- 上传视频
- 管理自己上传的视频
- 编辑视频信息
- 查看自己主页的投稿列表

### 4) Moderator
内容审核人员，可以：
- 审核视频
- 隐藏或删除违规评论
- 处理内容状态

### 5) Admin
管理员，可以：
- 管理用户
- 管理视频
- 管理评论
- 管理分类
- 管理推荐位
- 查看后台数据和审计记录（后续可扩展）

## 4.1 In Scope
本阶段必须实现：

1. 用户注册 / 登录 / 登出
2. 用户基础资料
3. 视频上传
4. 视频转码处理

## 6. Key User Flows

## 6.1 User Registration & Login Flow
1. 用户进入登录 / 注册页面
2. 输入用户名 / 邮箱 / 密码
3. 完成注册
4. 登录后获取用户身份
5. 页面刷新后仍可保持登录态

## 6.2 Video Upload Flow
1. 创作者进入投稿页面
2. 填写标题、简介、分类、标签等基础信息
3. 选择视频文件上传
4. 前端进行分片上传
5. 上传成功后后端创建或更新视频记录
6. 系统异步发起转码任务
7. 视频进入待处理 / 待审核状态
8. 审核通过后视频对外发布

## 6.3 Video Consumption Flow
1. 用户进入首页
2. 浏览推荐视频或分类视频
3. 点击进入视频详情页
4. 播放视频
5. 查看视频信息与作者信息
6. 评论、点赞、收藏

## 6.4 Admin Review Flow
1. 管理员进入后台
2. 查看待审核视频列表
3. 审核视频内容
4. 将视频标记为通过 / 拒绝 / 下架
5. 审核结果写入状态记录

## 8. Functional Requirements

## 8.1 Authentication

### Features
- 用户注册
- 用户登录
- 用户登出
- 获取当前登录用户
- 用户基础身份识别

### MVP Constraints
- 统一采用 JWT；登录颁发 token，可用 httpOnly Cookie + Header 兼容。
- 优先支持用户名 / 邮箱 + 密码登录
- 不支持第三方登录
- 不支持找回密码（可后续扩展）

### Acceptance Criteria
- 未登录用户可访问公开页面
- 登录后可访问评论、收藏、投稿等功能
- 退出登录后受限功能不可再访问
- 后台页面必须受角色权限保护

---

## 8.2 User Profile

### Features
- 个人头像
- 用户昵称
- 用户简介
- 个人投稿列表

### MVP Constraints
- 仅支持基础资料编辑
- 暂不支持复杂主页装修
- 暂不支持关注 / 粉丝列表
- `creator` 角色由后台授予，投稿/上传需具备该角色。

### Acceptance Criteria
- 用户可以查看自己的主页
- 其他用户可以查看公开主页
- 主页可以展示该用户已发布视频列表

---

## 8.3 Video Upload

### Features
- 创建视频稿件
- 填写标题 / 简介 / 分类 / 标签
- 上传视频文件
- 展示上传进度
- 上传完成后触发后端处理流程

### MVP Constraints
- 单视频投稿
- 不支持合集 / 分P
- 不支持高级视频编辑
- 不支持字幕文件上传

### Acceptance Criteria
- 创作者可以创建稿件
- 支持上传视频文件
- 上传成功后可以看到稿件状态
- 上传失败时有明确提示
- 上传后系统会进入后续处理流程

---

## 8.4 Video Processing

### Features
- 视频元数据提取
- 自动封面生成
- 视频转码
- 视频状态更新

### MVP Constraints
- 转码为基础播放格式即可
- 暂不支持复杂多码率策略
- 暂不支持手动转码配置
- 上传完成后必须经过审核，流程：draft/uploading → uploaded → transcoding → review_pending → published/rejected/hidden/deleted。

### Acceptance Criteria
- 视频上传后能进入转码流程
- 转码成功后可播放
- 转码失败时可被管理员或创作者识别

---

## 8.5 Video Playback Page

### Features
- 播放器展示
- 标题与简介展示
- 作者信息展示
- 点赞 / 收藏
- 评论区
- 推荐视频列表

### MVP Constraints
- 暂不支持弹幕
- 暂不支持投币 / 充电
- 暂不支持多线路切换
- 仅已发布视频对前台公开；作者/管理员可查看自身非公开状态。

### Acceptance Criteria
- 用户可流畅播放公开视频
- 页面可展示基础视频信息
- 登录用户可以评论、点赞、收藏
- 页面可展示相关推荐内容

---

## 8.6 Comments

### Features
- 发布评论
- 回复评论
- 删除自己的评论
- 后台隐藏或删除评论

### MVP Constraints
- 支持两级评论结构
- 不支持表情包和富文本
- 不支持@用户
- 不支持置顶评论
- 默认公开发布，后台/管理员可隐藏或删除。

### Acceptance Criteria
- 登录用户可以发表评论
- 评论能展示作者、时间和内容
- 支持一级评论和二级回复
- 管理员可处理违规评论

---

## 8.7 Like / Favorite

### Features
- 点赞视频
- 取消点赞
- 收藏视频
- 取消收藏

### MVP Constraints
- 仅支持视频维度点赞 / 收藏
- 暂不支持评论点赞
- 暂不支持收藏夹分组

### Acceptance Criteria
- 登录用户可对视频点赞和收藏
- 用户可取消操作
- 页面状态可正确同步

---

## 8.8 Homepage

### Features
- 推荐视频流
- 最新发布
- 热门内容
- 分类入口
- 后台可配置推荐位（简单权重/置顶）。

### MVP Constraints
- 推荐策略采用简单排序（最新/热门）+ 后台推荐位配置
- 不做个性化推荐
- 不做瀑布流复杂交互

### Acceptance Criteria
- 首页可以展示视频列表
- 用户可以从首页进入播放页
- 首页内容支持按分类过滤或跳转

---

## 8.9 Categories

### Features
- 分类列表
- 分类页视频展示
- 视频归属分类

### MVP Constraints
- 分类数量有限
- 分类由后台配置
- 暂不支持复杂子分类体系
- 搜索可选按分类过滤。

### Acceptance Criteria
- 视频可归属到某个分类
- 用户可按分类浏览视频
- 后台可管理分类

---

## 8.10 Admin Panel

### Features
- 管理员登录后可访问后台
- 视频管理
- 评论管理
- 用户管理
- 分类管理
- 审核管理
- 推荐位简单配置
- 授予/收回 `creator` 角色

### MVP Constraints
- 后台只做基础内容管理
- 不做复杂 BI 看板
- 不做复杂运营系统

### Acceptance Criteria
- 管理员可查看和管理内容
- 管理员可审核视频
- 管理员可处理评论
- 可配置推荐位
- 可授予创作者角色
- 非管理员不可访问后台页面

## 13. MVP UX Requirements

## 13.1 Basic UX Standards
- 页面必须有 loading 状态
- 页面必须有空状态
- 页面必须有错误提示
- 所有高风险操作必须有确认提示
- 所有表单必须有校验反馈（zod + react-hook-form）
- 所有异步操作必须有处理中反馈

## 13.2 Player UX Standards
- 支持基础播放控制
- 支持全屏
- 支持倍速（建议）
- 支持播放进度记忆（建议）
- 播放错误时有明确提示

## 13.3 Upload UX Standards
- 能看到上传进度
- 能看到处理状态
- 失败时能重新操作
- 不要求一次性极致体验，但必须流程完整清晰

---

## 14. Non-Functional Requirements

## 14.1 Performance
MVP 阶段要求：
- 首页初始加载可接受
- 视频播放页可稳定打开
- 上传过程不中断
- 转码任务不阻塞主站点请求

## 14.2 Reliability
- 上传记录可追踪
- 转码失败可识别
- 关键流程应有日志
- 后台操作应有基本审计能力

## 14.3 Security
- 需要登录的功能必须鉴权
- 后台功能必须角色校验
- 上传文件必须限制类型与大小
- 用户输入必须经过校验
- 不在前端暴露敏感信息

---

## 17. Open Questions

以下问题在开发前或开发过程中需要明确：

1. 登录方案最终采用 JWT 还是 session
2. Creator 角色是注册即拥有，还是需要后台授予
3. 视频审核是否默认开启，还是 MVP 阶段允许直发
4. 首页推荐是否完全基于排序，还是允许后台配置推荐位
5. 评论是否先上审核，还是先默认公开
6. 上传是否限制视频时长与大小
7. 是否保留原视频文件
8. MVP 是否要支持简单搜索入口

## 18. Agent Implementation Notes

AI agent 在实现本产品时必须遵循：

1. 优先实现 MVP 范围内功能，不提前扩展非目标功能
2. 所有页面必须考虑 loading / empty / error 状态
3. 所有后端接口必须校验输入
4. 所有后台页面必须做权限保护
5. 视频发布链路必须遵守状态流转
6. 上传、转码、审核、播放必须视为同一主链路进行设计
7. 不得擅自增加弹幕、直播、支付等超出 MVP 范围的能力
8. 如发现需求不明确，先提出问题而不是自行假设

---

## 19. Summary

本产品是一个以“视频上传与播放闭环”为核心的内容平台 MVP。  
当前阶段重点是完成从内容创作到内容消费的主路径，并通过最小必要功能支撑用户互动与后台管理。

产品建设优先级：
1. 主链路完整
2. 核心页面可用
3. 状态流转清晰
4. 后台可管理
5. 为后续扩展保留结构空间
